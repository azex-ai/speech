**Title**: 做了个 Mac 语音输入工具，专门解决说英文术语发音不标准的问题

**Body**:

最近在做一个 Mac 语音输入工具，分享一下思路，想听听大家的反馈。

## 背景

我做 Crypto 和 AI 相关的工作，每天要说大量英文术语。但我英语发音不标准（相信很多人都是），导致：

- 苹果听写把 "EigenLayer" 识别成乱七八糟的东西
- Whisper 系的工具好一点，但 Crypto 术语还是经常错
- 中英混合输入更是灾难——说中文夹一个英文术语，两边都识别不好

试了市面上的几个工具（Whisper Transcription、Superwhisper、voxt 等），要么不支持中英混合，要么对专业术语的识别太差。

## 思路

核心思路不是去微调 ASR 模型（成本高，泛化差），而是在识别后做后处理纠正。我设计了四层纠正系统：

1. **个人词库**（最高优先级）——你每次纠正浮窗里的识别结果，app 就记住这个映射。用得越多越准。
2. **上下文词**——通过 macOS 无障碍 API 读取当前活动窗口的内容，提取专业术语作为识别提示。你在看 DeFiLlama 的时候，它就知道你可能会说 TVL、Solana。
3. **领域词典**——内置了 Crypto 和 AI 两个术语纠正词典，几百个常见错误映射。
4. **校准**——第一次启动时朗读一段领域相关的句子，系统对比 ASR 输出和标准文本，自动生成你的个人发音映射。

## 技术选型

- ASR 引擎：sherpa-onnx + Paraformer-zh（中文和中英混合场景下开源最强）
- 语言：Swift 6 + SwiftUI，纯原生 Mac app
- 音频：AVAudioEngine，16kHz mono Float32
- 快捷键：右 Option 按下开始录音，松开结束（也可以自定义）
- 文本输入：pbcopy + CGEvent Cmd+V（最稳定的跨 app 方案）
- 数据存储：纯 JSON 文件，不用数据库
- 全部本地运行，不联网，不传数据

一个有意思的技术发现：Paraformer 自带的热词注入机制（SeACo-CTC）其实和 Paraformer 架构不兼容，所以后处理替换不是"退而求其次"，而是目前正确的技术路线。

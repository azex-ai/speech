## Submission checklist

Before submitting PRs:

- [ ] Ensure the project has a proper README with screenshots
- [ ] Verify the project builds and runs successfully
- [ ] Check each awesome list's CONTRIBUTING.md for specific requirements
- [ ] Follow alphabetical ordering where the list uses it
- [ ] One PR per awesome list, small and focused
- [ ] PR description should explain why the project fits the list
- [ ] Wait at least a week between submissions to avoid looking spammy
